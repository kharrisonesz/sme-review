USB & File Management
▼
USB setup and file compression by manufacturer
📄 In-Clinic Workflow📖 WG pp.25–26
Prepare your files before uploading:
  • One USB per manufacturer recommended (sharing is OK with proper file management)
  • Abbott: PDF folder + log files zipped together
  • Boston Scientific: entire BSC file zipped
  • Biotronik: ICS3000 folder and all listed files
  • Medtronic: PDFs zipped (tablet .pkg files are already compressed)
  • Always compress from the furthest point in the file structure
Clinics can share USBs across manufacturers with proper file management between uses.
Upload Process
▼
Upload clinic files to PaceMateLIVE™
🎓 In-Clinic CIED Course📄 In-Clinic Workflow📖 WG pp.27–28
Complete upload workflow:
  • Select your clinic and location
  • Choose the zipped manufacturer file
  • Set date range (specific range or “all”)
  • Select reports to upload
  • Verify patient details (add DOB or remove special characters if needed)
  • Submit and confirm the green “Files sent for processing” banner
  • Allow processing time before reports appear in your worklist
Medtronic iPad uploads may require manually entering patient name, date of birth, or serial number.

In-Clinic Report Processing
▼
Navigate and configure in-clinic reports
🎓 In-Clinic CIED Course📖 WG p.30
Set up each in-clinic report for processing:
  • Filter your In-Clinic Worklist (tip: use “Uploaded by me”)
  • Open Report Details and select Provider and EMR Department
  • Choose the CPT Code or No Charge for each report
  • Select Location if applicable
  • Apply the 26 modifier when a manufacturer rep performed the interrogation
Complete evaluation and approve
🎓 In-Clinic CIED Course📖 WG pp.31–32
Finalize the in-clinic report:
  • Generate the evaluation form (“Generate Template” for preset text or “Evaluation Form” for free text)
  • Write your technical interpretation in the Observations box
  • Add a CSN for Epic-integrated clinics if applicable
  • Save & Approve to send to the e-signature board
If you have eSign permissions, Save & Approve automatically signs the report.
API/Integration Understanding (EMR Only)
▼
Skip this section if your implementation uses manual entry only (no EMR integration).
How EMR integration works
📖 WG pp.11, 14, 52
Key concepts for EMR-integrated clinics:
  • Demographics, medications, diagnosis codes, hospitalizations, and MRN pull automatically within 15 minutes
  • Never manually enter a Medical Record Number
  • If there’s a mismatch, delete the incorrect ID and resubmit to trigger automatic rematching
  • Submit a support ticket for repull requests if data doesn’t update
Patient mismatch causes and resolution
📄 EMR Sync Error Guide📖 WG pp.9, 50–52
Common causes: name variations between vendor and EMR, DOB discrepancies, MRN format differences. The EMR Sync Error Guide walks through resolution steps. Submit a support ticket for repull requests if data doesn’t update.
